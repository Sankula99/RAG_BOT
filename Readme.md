# 📄 RAG Document Assistant

A full-stack **Retrieval-Augmented Generation (RAG)** application that enables users to upload PDF documents and ask questions using natural language. The system retrieves relevant document context through semantic vector search and generates context-aware responses using a locally hosted Large Language Model (LLM).

This project was built to explore modern RAG architectures, vector databases, embedding models, and end-to-end AI application development.

---

## Features

* Upload PDF documents for indexing
* Automatic document ingestion and text extraction
* Semantic text chunking and embedding generation
* Vector storage using Weaviate
* Natural language question answering over uploaded documents
* Context-aware responses generated by a locally hosted Ollama LLM
* Modern React frontend with FastAPI backend
* Source snippets returned alongside generated responses

## How It Works

### 1. Upload a PDF

The user uploads a PDF document through the React interface.

### 2. Document Processing

The backend:

* extracts text from the PDF
* splits text into overlapping chunks
* generates embeddings using Ollama
* stores vectors inside Weaviate

### 3. Ask Questions

When a user submits a question:

* the query is embedded
* the most relevant document chunks are retrieved using vector similarity search
* the retrieved context is passed to the LLM
* the generated answer is returned to the frontend

## Installation

### Prerequisites

* Python 3.x
* Node.js
* Docker
* Ollama

---

### Clone Repository

```bash
git clone https://github.com/yourusername/RAG-Document-Assistant.git

cd RAG-Document-Assistant
```

---

### Install Python Dependencies

```bash
pip install -r requirements.txt
```

---

### Install Frontend Dependencies

```bash
cd frontend

npm install
```

---

### Start Weaviate

```bash
docker-compose up -d
```

---

### Pull Required Ollama Models

Example:

```bash
ollama pull nomic-embed-text

ollama pull qwen2.5:3b
```

---

### Configure Environment

Create a `.env` file inside the `config` directory.

Example:

```env
OLLAMA_MODEL=qwen2.5:3b
EMBEDDING_MODEL=nomic-embed-text

weaviate_url=http://localhost:8080

chunk_size=1000
chunk_overlap=200
top_k=5

FILEPATH=./data/raw/
```

---

### Start Backend

```bash
uvicorn app.main:app --reload
```

Backend runs on:

```
http://localhost:8000
```

---

### Start Frontend

```bash
cd frontend

npm run dev
```

Frontend runs on:

```
http://localhost:5173
```

## API Endpoints

### Upload Document

```

```

```json
{
    "question": "What is Retrieval-Augmented Generation?"
}
```

Response
